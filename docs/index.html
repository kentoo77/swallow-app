凡例

難易度の目安：

  • かんたん 専門知識ほぼ不要。手順通りでOK
  • 注意 手順通りなら可能。エラー時は Cursor 等に相談
  • 要確認 専門知識があると安心。つまずいたらエンジニアに相談

1準備フェーズ

Step 1：Apple Developer Program に登録 かんたん

https://developer.apple.com/programs/

  • 年額 $99
  • 審査に数日かかる場合あり。先にやっておくと後が楽

Step 2：Google Play 開発者アカウントに登録（Android を出す場合） かんたん

https://play.google.com/console/signup

  • 初回 $25（一度きり）
  • 本人確認が必要

Step 3：必要なツールをインストール かんたん

  • Node.js：https://nodejs.org
  • Xcode：Mac App Store から（無料）
  • Android Studio（Android を出す場合）：https://developer.android.com/studio

2Capacitor セットアップ

Step 4：Capacitor プロジェクトを初期化 注意

cd 教材/swallow npm init -y npm install @capacitor/core @capacitor/cli @capacitor/ios @capacitor/android npx cap init Swallow com.masaenglishcompany.swallow

エラーが出たら、エラーメッセージを Cursor に貼って相談。

Step 5：Web アプリを www に配置 注意

mkdir -p www cp index.html www/

アセット（動画・画像）がある場合：

  • masa_avatar.png、masa1.MOV などの動画ファイルも www/ にコピー
  • index.html と同じ階層に置く（相対パスで参照しているため）
# 例：swallow フォルダ内に動画・画像がある場合 cp *.MOV www/ 2>/dev/null || true cp *.png www/ 2>/dev/null || true

Step 6：capacitor.config.json を確認 かんたん

webDir が www を指していることを確認。

{ "appId": "com.masaenglishcompany.swallow", "appName": "Swallow", "webDir": "www", "server": { "androidScheme": "https" } }

Step 7：iOS プロジェクトを追加 かんたん

npx cap add ios

Step 8：Android プロジェクトを追加（Android を出す場合） かんたん

npx cap add android

Step 9：ビルド内容を同期 かんたん

npx cap sync

Web アプリを変更したら、毎回 npx cap sync を実行してから Xcode / Android Studio でビルドする。

3iOS リリース

Step 10：Xcode でプロジェクトを開く かんたん

npx cap open ios

Step 11：Xcode で署名設定 要確認

  • 左のプロジェクト一覧で「App」を選択
  • 「Signing & Capabilities」タブを開く
  • 「Team」で自分の Apple Developer アカウントを選択
  • Bundle Identifier が com.masaenglishcompany.swallow になっているか確認
つまずきやすいポイント
証明書・プロビジョニングプロファイルのエラーが出たら、エラーメッセージを Cursor に貼るか、エンジニアに相談。

Step 12：実機で動作確認（任意） かんたん

  • iPhone を USB で接続
  • 上部のデバイスで iPhone を選択
  • ▶ ボタンでビルド・実行

Step 13：アーカイブを作成 かんたん

メニュー「Product」→「Archive」。完了まで待つ。

Step 14：App Store Connect にアップロード 要確認

アーカイブ完了後、「Distribute App」→「App Store Connect」→「Upload」。

証明書エラーが出たら、エラーメッセージを確認して Cursor に相談。

Step 15：App Store Connect でアプリ情報を登録 かんたん

https://appstoreconnect.apple.com

  • 「マイ App」→「+」→ 新規アプリ作成
  • アプリ名、プライマリ言語、Bundle ID（com.masaenglishcompany.swallow）
  • スクリーンショット（iPhone 6.7", 6.5", 5.5" など）
  • 説明文、キーワード、プライバシーポリシー URL、カテゴリ、年齢制限

Step 16：審査に提出 かんたん

ビルドが処理されたら、そのビルドを選択して「審査に提出」。Apple の審査を待つ（通常 1〜数日）。

4Android リリース

Step 17：Android Studio でプロジェクトを開く かんたん

npx cap open android

Step 18：署名キーを作成 要確認

  • 「Build」→「Generate Signed Bundle / APK」
  • 「Android App Bundle」を選択
  • 「Create new...」でキーストアを作成
  • パスワード・エイリアス・有効期限を設定（パスワードは必ずメモ）
重要
キーストアのパスワードとファイルを紛失すると、アプリの更新ができなくなります。安全な場所にバックアップしてください。

Step 19：リリースビルドを作成 要確認

署名キーを選択してビルド。出力される .aab ファイルを保存。

ビルドエラーが出たら、エラーメッセージを Cursor に貼って相談。

Step 20：Google Play Console でアプリを作成 かんたん

https://play.google.com/console

「アプリを作成」→ アプリ名・デフォルト言語などを入力。

Step 21：ストア掲載情報を入力 かんたん

  • スクリーンショット（スマホ・タブレット用）
  • 説明文（短い・長い）
  • アプリアイコン
  • コンテンツレーティング（質問に答えて取得）
  • プライバシーポリシー URL
  • ターゲット層・広告の有無など

Step 22：AAB をアップロード かんたん

「リリース」→「本番」→「新しいリリースを作成」→ .aab をアップロード。

Step 23：審査に提出 かんたん

必要な項目がすべて緑になったら「審査に提出」。通常、数時間〜1日程度で審査完了。

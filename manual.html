アプリ概要

Swallow は、MasaEnglish が提供する英語会話学習用のモバイルアプリです。

Instagram 風の UI で、旅行・日常シーンの英会話を「カテゴリ → トピック → エピソード」の階層で学べます。スワイプで次々と会話カードを進める形式で、リスニング・シャドーイング・穴埋めクイズなどに対応しています。

主な機能

  • ホーム：ストリーク（連続学習日数）、今日のミッション、おすすめの先生
  • フォロー：フォロー中の先生（トピック）一覧、未学習バッジ
  • レッスン：会話カードのスワイプ、音声再生、穴埋めクイズ、文法解説
  • プロフィール：学習履歴、設定（フレーズ表示など）

データソース

重要： 会話データ（teachers 配列・phrases 配列）は index.html 内にハードコードされています。外部の data.js は参照していません。データの追加・編集は index.html 内の該当箇所を直接修正してください。

技術スタック

  • Web アプリ：HTML / CSS / JavaScript（単一ファイル）
  • ネイティブ化：Capacitor（Ionic 製）
  • 配信：iOS App Store（予定）

1準備フェーズ

1. Apple Developer Program に登録 大井手

https://developer.apple.com/programs/

  • 年額 $99
  • 審査に数日かかる場合あり。先にやっておくと後が楽

2. Google Play 開発者アカウントに登録（Android を出す場合） 大井手

https://play.google.com/console/signup

  • 初回 $25（一度きり）
  • 本人確認が必要

3. 必要なツールをインストール 大井手

  • Node.js（v18 以上推奨）：https://nodejs.org
  • Xcode（15 以上推奨）：Mac App Store
  • Android Studio（Android を出す場合）：https://developer.android.com/studio

バージョン要件： Capacitor v5 以上、Node.js v18+、Xcode 15+ を推奨。環境差分でハマる場合があるため、可能なら揃えてください。

2Capacitor セットアップ

4. Capacitor プロジェクトを初期化 Masa

cd 教材/swallow npm init -y npm install @capacitor/core @capacitor/cli @capacitor/ios @capacitor/android npx cap init Swallow com.masaenglishcompany.swallow

5. Web アプリを www に配置（動画・画像も必須） Masa

致命的： cp index.html www/ だけだとレッスン画面が真っ黒になります。アプリは masa1.MOV, masa2.MOV, miki1.MOV, masaki1.MOV など 8 本の動画と masa_avatar.png を参照しています。これらを www/ にコピーしてください。

mkdir -p www cp index.html www/ cp *.MOV www/ 2>/dev/null || true cp masa_avatar.png www/ 2>/dev/null || true # または：cp 教材/swallow/*.MOV www/ など、実際の配置に合わせて

コピー後、www/ 内で index.html と動画・画像が同じ階層にあるか確認。相対パス（masa1.MOV など）で正しく参照できること。

ファイルサイズ注意： .MOV は非圧縮に近く、1 本あたり数十〜数百 MB になることがあります。8 本をそのままバンドルすると App Store の OTA ダウンロード制限（200MB 以上は Wi-Fi 必須）に引っかかる可能性があります。MP4（H.264）への変換を検討してください。

5.5. アプリ化前の index.html 調整（必須） Masa

致命的： index.html の @media (min-width: 600px) にある .phone-frame（390×844px の枠、擬似ノッチ）はブラウザデモ用です。Capacitor の WebView では実機画面全体を使うため、iPad や横画面で「端末の中に端末の枠」が表示される異様な見た目になります。

対応： アプリ化前に .phone-frame のスタイル分岐を削除するか、実機表示用に調整してください。例：@media (min-width: 600px) 内の phone-frame 関連を削除、または max-width: 100% などで実機では全画面表示にする。

6. capacitor.config.json を確認・修正 Masa

npx cap init（手順 4）で自動生成された capacitor.config.json を開き、次の内容になっているか確認・修正してください。特に webDir が www であること。

{ "appId": "com.masaenglishcompany.swallow", "appName": "Swallow", "webDir": "www", "server": { "androidScheme": "https" } }

7. iOS プロジェクトを追加 Masa

npx cap add ios

8. Android プロジェクトを追加（Android を出す場合） Masa

npx cap add android

9. ビルド内容を同期 Masa

npx cap sync

9.5. 開発中の反復ワークフロー Masa

HTML やアセットを修正したら、毎回以下を実行してから Xcode で再ビルドしてください。

# www に変更をコピー cp index.html www/ cp *.MOV www/ 2>/dev/null || true cp *.png www/ 2>/dev/null || true # ネイティブプロジェクトに反映 npx cap copy ios # または npx cap sync ios

このサイクルがないと、修正のたびに手詰まりになります。

3iOS リリース

10. Xcode でプロジェクトを開く 大井手

npx cap open ios

11. Xcode で署名設定 大井手

  • 左のプロジェクト一覧で「App」を選択
  • 「Signing & Capabilities」タブ
  • 「Team」で自分の Apple Developer アカウントを選択
  • Bundle Identifier が com.masaenglishcompany.swallow になっているか確認

11.5. アプリアイコン設定（必須） 大井手

致命的： App Store 提出には 1024×1024px のアイコンが必須です。設定しないと Archive → Upload の段階でエラーになります。

  • Xcode で ios/App/App/Assets.xcassets/AppIcon.appiconset を開く
  • 1024×1024px の PNG を用意し、AppIcon にドラッグ＆ドロップ
  • または、appicon.co などで 1024px 画像から各サイズを生成して配置

12. 実機で動作確認（強く推奨） 大井手

Capacitor アプリは WebView の挙動がシミュレーターと実機で異なります。特に動画再生の autoplay 制限は実機で顕著です。動画中心のアプリのため、実機テストを強く推奨します。

  • iPhone を USB で接続
  • 上部のデバイスで iPhone を選択
  • ▶ ボタンでビルド・実行

13. アーカイブを作成 大井手

メニュー「Product」→「Archive」

完了まで待つ

14. App Store Connect にアップロード 大井手

アーカイブ完了後、「Distribute App」→「App Store Connect」→「Upload」

証明書・プロファイルは自動で処理される場合が多い

15. App Store Connect でアプリ情報を登録 大井手

https://appstoreconnect.apple.com

  • 「マイ App」→「+」→ 新規アプリ作成
  • アプリ名、プライマリ言語、Bundle ID（com.masaenglishcompany.swallow）、カテゴリ、年齢制限

プライバシーポリシー URL（必須）： App Store 提出に URL が必須です。「追加で必要な素材があれば」では作れません。Vercel の /privacy や GitHub Pages などに HTML をホストし、URL を用意してください。テンプレートは「アプリ プライバシーポリシー テンプレート」で検索可。

スクリーンショット： 必須サイズは iPhone 6.7"（1290×2796px）、6.5"（1242×2688px）、5.5"（1242×2208px）など。Xcode シミュレーターで該当デバイスを起動 → アプリを表示 → Cmd+S で保存。最低 3 枚以上、各サイズごとに用意。実機で撮る場合は解像度を確認。

16. 審査に提出 大井手

ビルドが処理されたら、そのビルドを選択して「審査に提出」

Apple の審査を待つ（通常 1〜数日）

補足・注意事項

大井手さん側で必要なもの

以下を事前に確認してください。揃っていないと進められません。

  • Mac（Xcode インストール済み）
  • Apple Developer アカウントへのアクセス権限
  • 実機の iPhone（動作確認用）

コンテンツ準備（審査通過に影響） Masa

phrases 配列に (編集してください) のプレースホルダーが残ったまま提出すると、Apple の Guideline 4.2「最低限の機能を満たしていない」でリジェクトされる可能性があります。提出前にコンテンツを完成させてください。

Google Fonts・オフライン Masa

アプリは Sora, DM Sans, Noto Sans JP を fonts.googleapis.com から読み込んでいます。オフラインでは文字化け（フォールバック）します。リリースするなら、フォントをローカルにバンドルするか、フォールバック挙動を確認してください。

iOS ステータスバー・セーフエリア Masa

ノッチ付き端末でヘッダーがステータスバーに被る可能性があります。@capacitor/status-bar プラグインの導入、または env(safe-area-inset-top) の適切な適用を検討してください。
